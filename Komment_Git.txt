git remote -v — это краткий вывод о подключенных удалённых репозиториях.
git remote show origin — более подробная информация о репозитории, 
    включая информацию о ветках.

    Если получилось так, что в коммит вошла директория,которая на ГитХаб не проходит 
        (большой объем), но в отслеживание она уже попала, то надо сделать следующее.
    1. Внести эту директорию вместе с вложениями (/ преред названием директории) 
        в .gitignore:

        /bert-base-cased_model_saved

    2. Т. к., она уже отслеживается (проиндекирована), то следует ее удалить из отслеживания
        (но чтобы она осталась в файловой системе):

        git rm -r --cached bert-base-cased_model_saved

            --cached — это означает, что файл/директория будет удалена только из индекса 
                        (не из вашего локального каталога).
            -r — используется для рекурсивного удаления всех файлов 
                        и поддиректорий внутри указанной директории.

    3. Надо удалить коммит (коммиты) которые уже были выполнены с включением 
        ненужной директории (иначе отправка в удаленный директорий с новым коммитом будет 
        начинаться с предыдущего куоммита, который включал эту директорию и процесс снова
        зависнит):

        git reset --soft HEAD~1

        где: 
            --soft — откатывает только коммит, оставляя изменения в файлах 
                (вы можете снова закоммитить эти изменения или отредактировать их).
            HEAD~1 — откатывает репозиторий на один коммит назад 
                (вы можете указать другое количество, если хотите откатиться 
                на несколько коммитов, например HEAD~2 для двух коммитов).
